  • Ant and Maven have occupied opposing positions:

Build Files in Groovy

  • XML is the wrong format for a build file.
  • Every Gradle build file is an executable Groovy script.
  • Because Gradle is based on Groovy, developers still have direct control of their builds.

Domain-Specific Build Languages

  • Scripting and Coding is discouraged
  • Gradle plug-ins
  • Gradle task definition
    • Even through the simple mechanism of a custom Gradle task, you can introduce small pieces of build vocabulary, turning an otherwise undesirable chunk of imperative code into a clean declaration.
  • Using and extending the Gradle DSL are usually preferable to writing code in your build file.

Task Configuration

  • A closure added to the task name without the left-shift operator doesn't create additional task action code. It is a configuration block.
  • Configuration blocks run during Gradle's configuration lifecycle phase - before the execution phase, when task actions are executed.
